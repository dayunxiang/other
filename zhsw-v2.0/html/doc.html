一些注意事项

css必须放在头部加载，脚本全部在</body>标签前引入,不允许在头部加载脚本！！！

所有页面的css样式都page.css中，所有页面的脚本都写在page.js中

所有的页面必须引入所有的css,不管该页面是否用到，js脚本、插件必须全部加载，不管当前页是否需要使用

公共样式采用【.p-xxx】，浮层中的样式采用【.i-xxx】,某个页面的样式采用【.page-xxx】

写脚本的时候记得添加约束，我一般的采用body的class作为第一个选择器来进行约束脚本的生效范围

如果开发们不是从body开始包前端代码，亲爱的，那么请你在body下面创建一个大的div来代替body也是一样的

如果页面有什么bug，请跟我说具体是【哪个静态页】！不要拿开发线上的页面墨迹半天，截图是最好的说明

亲爱的，做项目前麻烦养成先更新svn再干活的习惯，这样可以保持代码的同步和实时性

一般来说，只有修改了DOM结构的时候才会告知一下，更改css不会通知！！！

本次页面设计按照最小分辨1366*768，最大分辨率1920*1080，在IE8环境下采用纯色作为背景色

所有的子页面单独在浏览器浏览会出现白色背景，子页面必须嵌入框架页，因为浏览器不支持背景透明，会强制为白色

做项目不要赶，现在所赶出的进度都会在后期项目维护阶段一个不少的吐出来！！！

                                        $.fn.tab = function( options )
                                        {
                                            return $(this).each( function(){

                                                var _this = $( this ),
                                                    oDefaultOptions =
                                                    {
                                                        "tabNavClass"    : ".tab-nav",     //头部类名
                                                        "tabConClass"    : ".tab-con",     //内容类名
                                                        "activeClass"    : "active",       //当前状态类名
                                                        "disableClass"   : "disable",      //禁用类名
                                                        "event"          : "click",        //选项卡事件触发类型
                                                        "init"           : null,           //初始化函数
                                                        "callBack"       : null            //回调函数
                                                    };

                                                oDefaultOptions = $.extend( oDefaultOptions, options || {} );

                                                // 触发前 初始化 只执行一次
                                                if( oDefaultOptions.init )
                                                {
                                                    oDefaultOptions.init();
                                                }

                                                _this.on( oDefaultOptions.event, oDefaultOptions.tabNavClass, function(){

                                                    if( $( this ).hasClass( oDefaultOptions.disableClass ) )
                                                    {
                                                        return false;
                                                    }

                                                    var nIndex = $( this ).index();

                                                    $( this ).addClass( oDefaultOptions.activeClass )
                                                             .siblings( oDefaultOptions.tabNavClass ).removeClass( oDefaultOptions.activeClass );

                                                    _this.find( oDefaultOptions.tabConClass ).removeClass( oDefaultOptions.activeClass )
                                                         .eq( nIndex ).addClass(oDefaultOptions.activeClass);

                                                    // 触发后 回调
                                                    if( oDefaultOptions.callBack )
                                                    {
                                                        oDefaultOptions.callBack({
                                                            dom:_this,
                                                            index:nIndex
                                                        });
                                                    }
                                                } );
                                            } );
                                        }
                                    }( jQuery );
                                    /**************** 选项卡定义 结束 */

滚动条插件

如果你的结构是 ul 包li,那么你只需要对ul这个层级使用滚动插件即可

一般来说，对某个容器使用滚动，就要对该容器设置一个固定的height值或者是max-height，否则滚动条是没办法出现的

layer弹窗插件有一个回调参数，就是当弹窗渲染完后要执行的函数， 如果弹窗内容比较多，layer会用原生的滚动条（比较丑）， 所以可以在回调函数中对弹窗的某个层级使用滚动条插件
